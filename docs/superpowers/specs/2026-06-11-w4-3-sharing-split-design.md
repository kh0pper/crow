## New tool-registration pattern (this repo has none — this slice sets it)

No in-repo server splits tool registration today (all servers register inline). The pattern this slice establishes, for future server splits to imitate:

```js
// servers/sharing/tools/<group>.js
export function register<Group>Tools(server, ctx) { /* server.tool(...) calls, verbatim bodies */ }
```

`ctx` is ONE shared object built in createSharingServer: `{ db, identity, peerManager, syncManager, instanceSyncManager, nostrManager, buildProjectCloneBundle, applyProjectCloneBundle, ... }` — built ONCE, passed to every register fn. Register fns are called in the exact order the tools were registered before.
