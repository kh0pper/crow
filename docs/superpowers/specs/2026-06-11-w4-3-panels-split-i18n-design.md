## Behavior-frozen contract

1. **Same exports:** each panel file keeps its default-export manifest (`id`, `name`, `icon`, `route`, `navOrder`, `category`, `hidden`, `preload`, `handler`) — the ONLY importer is `servers/gateway/dashboard/index.js` (:69/:75/:76, verified; no test or bundle imports these panels). The panel file becomes a thin orchestrator that imports from a new `panels/<panel>/` directory, exactly like the established `health.js` → `nest/`, `contacts.js` → `contacts/`, `messages.js` → `messages/` pattern.
2. **Same routes, same HTML:** for `lang=en`, rendered output must be byte-identical before/after (verified by the render-diff harness below). For `lang=es`, output changes ONLY where new translations replace previously hardcoded EN strings (the point of the i18n work).
3. **No logic edits** beyond (a) moving code, (b) wrapping user-facing string literals in `t()`/`tJs()`, (c) the import-path depth change inside extracted modules — `../shared/...` → `../../shared/...`, AND sibling/other-relative imports (`./bot-runtime-flag.js` → `../bot-runtime-flag.js`, `./nest/data-queries.js` → `../nest/data-queries.js`, `../../../db.js` → 4 levels — audit every import line AND every `__dirname`/`import.meta.url`-relative path: `LOCAL_REGISTRY` in extensions.js:25-26 joins `../../../../registry/add-ons.json` from `__dirname` and needs a fifth `../` plus a re-derived `__dirname` in its new file; a miss is silent at boot+suite — R3-3). No renames of functions, no signature changes beyond passing `lang`/`db` where an extracted function needs what it already closed over.
   **One blessed shape exception (review S6):** module-level display-label const maps whose VALUES are shown to users but whose KEYS carry the logic — `STATUS_LABEL` (bot-board.js:58) and `TABS` labels (bot-builder.js:70) — may become i18n-key maps resolved via `t(key, lang)` at use sites (or a tiny `labelFor(x, lang)` helper). Keys/ids stay frozen — they are routing/comparison values (TABS ids compared at :804/:807).
4. **i18n EN-exactness rule:** every new key's `en` value must EXACTLY equal the literal it replaces (including punctuation/whitespace). ES values are new translations following the existing tone in `i18n.js` (es is informal "tú" form per existing entries). `t()` falls back lang→en→raw key (i18n.js:989-1009), so a missing ES value renders EN — benign.
5. **DO-NOT-TRANSLATE list (review C5/C6 — these are data or logic, not copy):**
   - extensions.js privileged-install consent gate (:970 instruction, :974 placeholder): the typed token `INSTALL` is compared at :982 (`.toLowerCase() === "install"`). Translating instruction/placeholder without the comparison BRICKS privileged installs for ES users. Behavior-frozen ⇒ exclude the whole consent block from the sweep (leave EN; W3-4 can revisit).
   - Everything inside bot-builder `defaultDefinition()` (:321-363): it reads like prose but is persisted bot-definition DATA written to `pi_bot_defs` on create (:457). Wrapping it would persist Spanish prompts and change runtime behavior.
   - DB status values (`bot_sessions.status` compared at :1693-1696, `tasks_items` statuses, CARD_STATUSES) — data, not copy. Only display-label maps over them (see rule 3 exception) get translated.
   - bot-builder run-monitor tbody contents (:1698-1713): `#pibot-sessions-tbody` is replaced every 5s by an independent untranslated generator in `servers/gateway/routes/streams.js:250-292` (out of scope) — translating the panel-side tbody would flash ES→EN every 5s. Exclude tbody CONTENTS; the `<th>` headers and section title live outside the replace target and ARE swept.
   - **The whole `peer-edit.js` module and bot-board's `peer_toggle` branch (round-2 S-3 — byte-purity wins):** these are federation/boundary code moved byte-for-byte; their user-facing strings stay frozen EN this slice (W3-4 follow-up can translate after the verified move).

## Risks & guards

- **Import-depth bug class:** extracted modules import shared code via `../../shared/...`; a missed rewrite throws at import time → caught by suite + disposable boot. Each commit boots the gateway (disposable) before committing.
- **Template-literal escaping:** moving code between template strings can silently change `\`` or `${}` escaping — the EN render-diff is the guard.
- **`tJs` vs `t` context errors:** wrong choice breaks the client script (apostrophes). ES render + a manual click-through of the ES dashboard at deploy time; also the EN diff catches `t()`-in-JS regressions since `t`/`tJs` are EN-identical only when no escapable chars exist — reviewer must check each JS-context wrap uses `tJs`.
- **Parallel sessions:** NEVER stage `bundles/*` untracked dirs, `data`, `scripts/bench/results`, `servers/gateway/scheduler.js`. Positional-path commits only. The 1 standing suite failure (`tests/bundle-contract.test.js` registry drift) is parallel-owned — verify against clean main before attributing.
- **No DB/schema work in this slice.** No init-db change, no deploy-time migration. Deploy = pull + restart only.
